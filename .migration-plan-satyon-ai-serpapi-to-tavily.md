# Migration Plan: Add Tavily as parallel web search source alongside SerpAPI

**Unit ID:** satyon-ai-serpapi-to-tavily
**Search Provider:** serpapi
**Migration Strategy:** additive

## Objective

Introduce Tavily as an additional/configurable web search source in the SATYON-AI backend. The existing SerpAPI-based Google search remains intact; Tavily is added as a parallel source in the aggregator so both can be queried simultaneously (or Tavily can serve as fallback when SERPAPI_KEY is absent).

## Implementation Steps

1. Add tavily-python to backend/requirements.txt and root requirements.txt
2. Add TAVILY_API_KEY = os.getenv('TAVILY_API_KEY', '') to backend/config.py
3. Create backend/api_clients/tavily_api.py with an async search_tavily() function that calls the Tavily search API and returns results in the same dict shape as google_api.py ({title, url, snippet, source, source_icon}); include a demo fallback when no key is set
4. In backend/search_engine.py, import search_tavily, add 'tavily' to SOURCE_FUNCTIONS and get_available_sources(); add 'tavily' to source_priority dict in _score_and_sort()
5. Add TAVILY_API_KEY (sync: false) to render.yaml envVars

## Notes

SerpAPI is called directly via aiohttp (no google-search-results SDK), so the existing google_api.py is untouched. Tavily client should follow the same async aiohttp pattern or use the tavily-python SDK. The query_classifier.py source-selection logic may also need 'tavily' added to the general/web category if it gates which sources are activated automatically.
